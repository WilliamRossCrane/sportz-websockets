# 📄 Project Retrospective

## What Went Well

- Successfully implemented a **WebSocket-based communication layer** to enable real-time commentary updates.
- Designed the system to **handle structured JSON commentary data**, allowing messages to scale beyond simple text payloads.
- Implemented **mock data seeding** to simulate high-volume commentary streams and validate connection stability.
- Established a clear project structure with version control and meaningful commit messages.
- Instrumented the Node runtime with **APM monitoring** to observe performance and runtime behaviour.

---

## Challenges & Trade-offs

- Some of my error solving could have been better, I had to revert to AI or my teachers code to find issues.
- Learning how to have the server running across different browsers was quite difficult at first.

---

## Key Learnings

- WebSockets provide an efficient solution for **low-latency, real-time communication**, but require careful handling of connection lifecycle and message throughput.
- Observability tools (APM) are valuable for identifying **performance bottlenecks and runtime behaviour** in Node applications.
- Simulating realistic data loads is important for validating system reliability before production use.

---

- Using **Git and conventional commits** for clear project history
